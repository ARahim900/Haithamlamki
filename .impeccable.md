# Abraj MIS Dashboard — Design Context

## Users

**Primary audiences** (all equally important):

- **Rig Managers** — Desktop in office. Reviewing daily DDOR reports, approving operations, analyzing fleet performance. Long sessions, data-dense workflows.
- **Field Engineers** — Tablets on-site at rig locations. Entering DDOR data, fuel consumption, well tracking in harsh lighting conditions. Quick data entry, large touch targets, minimal scrolling.
- **Senior Management** — Mobile or meeting-room screens. Glancing at fleet KPIs, revenue vs budget, utilization trends. Need instant clarity from summary views.

**Jobs to be done:**
1. Enter and approve daily drilling operations reports (DDOR) accurately
2. Track fleet status, utilization, and revenue across 30+ rigs in real time
3. Identify problems early — NPT spikes, budget overruns, CSAT drops
4. Generate billing tickets and financial reconciliation

**Context:** Oil & gas drilling operations in Oman. High-stakes environment where data accuracy directly impacts safety, billing, and client relationships with operators like PDO, OXY, and BP.

## Brand Personality

**Three words:** Modern, Sharp, Performance-Driven

**Tone:** A modern energy company that competes with digital-first organizations, combined with operational excellence where every number matters. This is not a generic admin panel — it should feel like a purpose-built command center for drilling operations.

**Emotional goals** (all four apply):
- **Confident & in control** — "I can see everything, nothing is slipping through the cracks"
- **Efficient & fast** — "I get in, do my job, get out — no friction"
- **Professional & trustworthy** — "This is a serious tool I can rely on for accurate data"
- **Calm & organized** — "Even with 30 rigs, the information feels manageable"

## Aesthetic Direction

**Visual tone:** Industrial precision meets modern tech. Clean lines, data-dense but not cluttered. Information hierarchy is king — the most critical numbers (active rigs, NPT, revenue) should be immediately scannable.

**Brand colors:**
- Primary: Abraj Teal `#2D7A89` — the anchor color
- Dark: `#243B42` — sidebar, headers
- Accent: `#00D2B3` — highlights, CTAs
- Supporting chart palette: Blue `#5B8DB8`, Amber `#D4A24C`, Sage `#3A9E7E`, Rose `#B06B6F`, Dark Teal `#1E5A66`

**Typography:** Inter — clean, professional, excellent tabular figures for data-heavy tables.

**Theme:** Light mode primary + dark mode support (for field use in varying lighting conditions).

**Anti-references:** Do NOT look like:
- Generic SaaS dashboards with gradient cards and glassmorphism
- Consumer apps with playful rounded elements
- Over-designed analytics tools with decorative charts that obscure data

**References:** The feel of Bloomberg Terminal's information density, Figma's clean professionalism, Linear's operational sharpness — adapted for industrial/energy context.

## Design Principles

1. **Data first, decoration never.** Every pixel should serve the operator. Charts, tables, and KPIs exist to inform decisions — not to impress. If a visual element doesn't help someone do their job, remove it.

2. **Scannable in 3 seconds.** A rig manager glancing at the dashboard during a morning meeting should grasp fleet status instantly. Use color coding, spatial hierarchy, and consistent patterns so the eye knows where to look.

3. **Touch-ready, desk-optimized.** Design for the hardest context first (field engineer on a dusty tablet in sunlight), then enhance for the easiest (desktop with mouse). 44px minimum touch targets. 16px minimum input font. No hover-dependent interactions for core functionality.

4. **Trust through consistency.** The same color always means the same thing: green = operating/good, amber = warning/reduced, red = NPT/critical, gray = inactive/stacked. Deviate from this and users lose confidence.

5. **Respect the operator's time.** Field engineers entering DDORs at 5 AM don't want friction. Minimize clicks, pre-fill what you can, auto-calculate what's derivable. Every unnecessary step is a cost.

## Accessibility Requirements

- **WCAG AA compliance** — minimum 4.5:1 contrast for normal text, 3:1 for large text
- **Full keyboard navigation** — all interactive elements reachable via Tab, operable via Enter/Space
- **Reduced motion support** — respect `prefers-reduced-motion`
- **Color-blind safe** — never use color alone to convey meaning; pair with icons, labels, or patterns
- **Dark mode** — full dark theme for field use and user preference

## Technical Stack

- Next.js 15.5 / React 19 / TypeScript
- Tailwind CSS v4 + custom CSS (dashboard.css)
- Recharts for data visualization
- Supabase (PostgreSQL) for persistence
- Motion (Framer Motion) for animations
- Lucide React for icons
- Geist font family available (currently using Inter)
